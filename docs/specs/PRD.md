# PRD: 表白墙/祝福墙网站

## Problem Statement

校园/公司举办活动时（如毕业季、情人节、节日庆祝、员工生日等），需要一个线上平台让大家留下祝福和表白。目前缺少一个轻量、免费、国内可稳定访问的多墙祝福平台。每个活动可以创建一面独立的墙，参与者通过链接访问、留言、点赞，管理员可以管理内容。

## Solution

基于 LeanCloud 国内版构建一个多墙祝福平台：

- **多墙模式**：每个活动创建一面独立的祝福墙，通过唯一链接访问
- **轻量前端**：React + TypeScript + Tailwind CSS，部署在 LeanCloud 静态托管
- **零后端**：直接通过 LeanCloud JS SDK 读写数据，无需自建后端
- **极低成本**：LeanCloud 免费层覆盖中小型活动（几百到几千条留言）

## User Stories

### 管理员系统

1. As a 管理员, I want to 使用固定的账号密码登录管理后台, so that 只有授权人可以管理
2. As a 管理员, I want to 创建一面新的祝福墙并自定义标题和封面描述, so that 参与者知道这是一面什么活动的墙
3. As a 管理员, I want to 设置墙的主题色, so that 视觉上与活动氛围匹配
4. As a 管理员, I want to 获得一个唯一的分享链接, so that 我可以把它发给参与者
5. As a 管理员, I want to 在管理面板中查看所有墙及各墙的留言, so that 统一管理
6. As a 管理员, I want to 删除某条留言（需二次确认）, so that 管理不当内容
7. As a 管理员, I want to 删除整面墙及其所有留言（需二次确认）, so that 清理过期活动
8. As a 管理员, I want to 设置墙为"已关闭"状态, so that 活动结束后不再接受新留言但仍可浏览
9. As a 管理员, I want to 设置墙为"隐藏"状态, so that 从首页列表中隐藏但通过链接仍可访问
10. As a 管理员, I want to 恢复已关闭或已隐藏的墙的状态, so that 灵活管理
11. As a 管理员, I want to 查看每面墙的统计数据（留言数、点赞总数）, so that 了解活动参与情况

### 参与者浏览

12. As a 参与者, I want to 通过链接直接打开一面祝福墙, so that 不需要注册登录
13. As a 参与者, I want to 以等高卡片网格形式浏览所有祝福（按最新排序）, so that 视觉整齐、信息密度高
14. As a 参与者, I want to 点赞我喜欢的祝福, so that 表达对某条留言的认可
15. As a 参与者, I want to 看到每条祝福的点赞数, so that 感知哪些内容受欢迎
16. As a 参与者, I want to 在墙关闭后仍可浏览已有留言, so that 回顾活动内容
17. As a 参与者, I want to 将某条留言卡片下载为图片保存到本地, so that 收藏喜欢的祝福

### 参与者留言

18. As a 参与者, I want to 填写昵称和祝福内容来发布一条留言, so that 留下我的祝福
19. As a 参与者, I want to 留空昵称时自动以"匿名"身份发布, so that 可以匿名留言
20. As a 参与者, I want to 在祝福内容中使用 emoji（通过系统键盘输入）, so that 表达更丰富的情感
21. As a 参与者, I want to 发布后立即在墙上看到我的留言, so that 确认发送成功
22. As a 参与者, I want to 发布留言时有基本的输入校验（内容不能为空、最大 1000 字）, so that 避免垃圾内容
23. As a 参与者, I want to 在墙关闭后无法发布新留言, so that 符合活动结束的预期

### 全局

24. As a 访客, I want to 在首页看到所有公开的祝福墙列表, so that 发现和浏览不同的活动墙
25. As a 访客, I want to 在墙列表超过 20 个时看到搜索框, so that 快速找到想要的墙
26. As a 访客, I want to 页面在手机上也有良好的浏览体验, so that 随时随地参与

## Implementation Decisions

### 技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| 前端框架 | React 18 + TypeScript | 生态成熟 |
| UI 样式 | Tailwind CSS | 轻量、快速出活 |
| 构建工具 | Vite | 开发体验好 |
| 数据层 | LeanCloud JS SDK | 免后端，直接前端读写 |
| 部署 | LeanCloud 静态托管 | 一站式，国内访问快 |
| 路由 | React Router | SPA 路由 |
| 卡片图片生成 | html2canvas | 将留言卡片渲染为图片下载 |

### 数据模型

**Wall（祝福墙）**

| 字段 | 类型 | 说明 |
|------|------|------|
| title | String | 墙标题 |
| description | String | 墙描述/副标题 |
| protagonist | String | 祝福送给谁（主角），空则默认为墙标题 |
| themeColor | String | 主题色（hex） |
| slug | String | URL 友好标识，唯一索引 |
| status | String | 状态：`open` / `closed` / `hidden` |
| createdAt | Date | 创建时间 |
| likeCount | Number | 总点赞数（冗余统计） |
| messageCount | Number | 留言总数（冗余统计） |

- `open`：正常状态，可浏览可留言
- `closed`：已关闭，可浏览不可留言，墙详情页显示"已关闭"提示
- `hidden`：已隐藏，从首页列表中不显示，但通过链接仍可访问和留言

**Message（留言）**

| 字段 | 类型 | 说明 |
|------|------|------|
| wall | Pointer<Wall> | 所属墙 |
| nickname | String | 昵称，空则为"匿名" |
| content | String | 祝福内容（纯文本 + emoji），最大 1000 字 |
| likeCount | Number | 点赞数 |
| createdAt | Date | 发布时间 |

**Like（点赞）**

| 字段 | 类型 | 说明 |
|------|------|------|
| message | Pointer<Message> | 所属留言 |
| fingerprint | String | 访客标识（基于 localStorage + IP 组合） |
| createdAt | Date | 点赞时间 |

**AdminConfig（管理员配置）**

| 字段 | 类型 | 说明 |
|------|------|------|
| username | String | 管理员用户名 |
| passwordHash | String | 密码哈希值 |
| key | String | 固定 key，用于查找配置 |

> 管理员登录通过 LeanCloud 云函数进行密码哈希比对，前端不处理密码逻辑。

### 页面路由

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | 首页 | 公开墙列表（>20 个时显示搜索框） |
| `/login` | 管理员登录 | 管理员输入账号密码 |
| `/admin` | 管理面板 | 所有墙的列表及管理操作（需登录） |
| `/admin/walls/new` | 创建墙 | 管理员创建新墙 |
| `/w/:slug` | 墙详情 | 展示留言卡片 + 发布入口 + 下载卡片功能 |

### 卡片展示设计

- 采用**等高卡片网格布局**（响应式：手机 1 列、平板 2 列、桌面 3-4 列）
- 每张卡片显示：祝福内容 + 昵称（或"匿名"）+ 发布时间 + 点赞按钮及数量 + 下载按钮
- 卡片颜色/装饰跟随墙的主题色
- 留言按最新发布时间倒序排列

### 卡片下载为图片

- 弹窗采用**左右布局**：左侧模板选择（3 列网格），右侧实时预览
- 使用 Canvas 2D API 绘制精美贺卡模板，下载为 PNG
- 内置模板风格：星空许愿（深蓝星空）、月夜情书（米色手绘）、温馨信纸（横线信纸）等
- 收件人自动取自墙的 `protagonist` 字段，无需手动输入
- 预览随模板切换实时更新

### 防刷机制

- 点赞基于浏览器 fingerprint（localStorage + IP 组合），同一用户对同一条留言只能点一次赞
- 发布留言通过 LeanCloud 云函数限频：同一 IP 每分钟最多 5 条
- 留言内容限制最大 1000 字

### 匿名机制

- 昵称字段为空时，前端显示"匿名"
- 后端不做限制，匿名与否完全由用户选择

### 删除二次确认

- 删除留言和删除整面墙均需弹出确认弹窗
- 确认弹窗明确提示操作后果（如"删除后不可恢复"）

### 墙状态说明

| 状态 | 首页列表 | 通过链接访问 | 留言 |
|------|---------|------------|------|
| `open` | ✅ 显示 | ✅ 正常 | ✅ 可留言 |
| `closed` | ✅ 显示 | ✅ 正常（提示"已关闭"） | ❌ 不可留言 |
| `hidden` | ❌ 不显示 | ✅ 正常 | ✅ 可留言 |

## Testing Decisions

- 测试重点：核心交互逻辑（管理员登录、发布留言、点赞、管理删除、状态切换）
- 关键测试场景：
  - 空昵称时正确显示"匿名"
  - 同一用户不能重复点赞
  - 墙关闭后不能发布新留言但可浏览
  - 墙隐藏后不在首页显示但链接可访问
  - 管理员密码错误时拒绝登录
  - 留言内容为空或超 1000 字时提示校验
  - 删除操作弹出二次确认弹窗
  - 卡片下载功能生成包含完整信息的图片
  - 同一 IP 每分钟发布超过 5 条时被限制

## Out of Scope

v1 不包含以下功能，后续可迭代添加：

- 图片/视频上传（v1 只支持文字 + emoji）
- 普通用户注册/登录系统
- AI 辅助写祝福文案
- 定时揭晓功能
- 多语言支持
- 留言评论/回复（嵌套）
- 消息通知
- 墙的访问密码保护
- 数据导出
- 主题模板选择（v1 仅支持主题色）
- 留言分页/加载更多
- 二维码生成
- 留言排序切换（v1 固定最新在前）
- 敏感词过滤

## Further Notes

- LeanCloud 国内版需要绑定已备案的域名才能使用自定义域名，初始阶段使用 LeanCloud 提供的子域名即可
- LeanCloud 免费层限制：API 调用 3 万次/天，存储 1GB，并发连接 100。对于中小型活动足够
- 如果活动突然爆发流量（如万级并发），需要升级 LeanCloud 付费套餐
- 管理员密码的哈希校验通过 LeanCloud 云函数实现，避免在前端暴露逻辑
- emoji 支持通过系统原生 emoji 键盘输入即可，不需要额外的 emoji 选择器组件
- 管理员账号密码在 LeanCloud 云函数中初始化配置（部署时设置），不通过前端注册
